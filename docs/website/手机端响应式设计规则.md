# 手机端响应式设计规则

## Responsive Design Rules
Same content, same URLs, responsive layout, mobile-first CTA priority.

### 1. Build mobile-first

采用移动端优先的开发策略，先确保移动设备上的用户体验，再逐步扩展到桌面端。

### 2. Use the same URL and same content for mobile and desktop

移动端和桌面端使用相同的URL和内容，避免为不同设备创建独立的页面。

### 3. Do not create separate mobile pages

不创建独立的移动端页面，通过响应式设计实现跨设备适配。

### 4. Desktop may use multi-column layouts

桌面端可采用多栏布局，充分利用较大的屏幕空间。

### 5. Mobile must use single-column layouts

移动端必须使用单列布局，确保在小屏幕上的可读性和易用性。

### 6. Primary CTA must appear above the fold on mobile

主要的行动号召按钮（CTA）必须在移动端首屏可见区域内显示。

### 7. Add sticky bottom CTA on mobile for key conversion pages

在关键转化页面的移动端添加固定底部CTA按钮，提升转化率。

### 8. Use accordions for FAQ on mobile

在移动端使用手风琴组件展示FAQ内容，节省空间并提升用户体验。

### 9. Avoid long paragraphs on mobile

避免在移动端使用过长的段落，保持内容简洁易读。

### 10. Test all pages at 375px, 390px, 768px, 1024px, and desktop width

在以下宽度测试所有页面：
- 375px（iPhone SE/旧款iPhone）
- 390px（iPhone 12/13/14系列）
- 768px（iPad竖屏）
- 1024px（iPad横屏）
- 桌面端宽度
